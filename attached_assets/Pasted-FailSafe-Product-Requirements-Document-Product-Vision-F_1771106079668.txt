FailSafe - Product Requirements Document
Product Vision
FailSafe is a web application that transforms failures into connections. Users share setbacks anonymously, get matched with others facing similar challenges, and support each other through accountability partnerships and daily challenges. The experience emphasizes uplift, growth, and celebrating small wins.
Tech Stack
Required

Frontend: React 18+ with Tailwind CSS
Backend: Node.js with Express
Database: Convex.dev (real-time database)
AI: MiniMax API for challenge generation and suggestions
Real-time Chat: Socket.io
Deployment: Replit

Optional Tools (if available credits)

Bem, Daytona, Runloop for development workflow

Design Principles

Clean, minimal interface with intuitive navigation
Warm, supportive color palette (soft purples #8B7AB8, warm oranges #FF9F66, calming blues #7EC8E3)
Anonymous-first design to reduce judgment and increase authenticity
Mobile-responsive from day one

MiniMax API Integration Requirements
1. Daily Challenge Generation
Endpoint: /api/challenges/generate
Input:
json{
  "chatRoomId": "room_123",
  "users": [
    {
      "username": "Phoenix_4521",
      "category": "Job Search",
      "failures": ["Failed interviews", "No responses"],
      "goal": "Land a PM role"
    }
  ]
}
MiniMax Prompt:
"Given that these users failed at [job interviews] and their goal is [landing a PM role], suggest ONE small, specific, actionable challenge they can do together TODAY. Keep it under 30 words. Make it collaborative and achievable in 15-30 minutes."
Output:
json{
  "challenge": "Do one mock interview together and give each other feedback on one specific improvement",
  "estimatedTime": 30
}
2. Encouragement Generator
Endpoint: /api/encouragement/generate
Input:
json{
  "failureCategory": "Job Search",
  "failureDescription": "Bombed my dream job interview"
}
MiniMax Prompt:
"Write a short, genuine, encouraging message (under 25 words) for someone who failed at [job interview]. Be warm but not toxic-positive. Acknowledge the difficulty."
Output:
json{
  "message": "Interviews are tough. Every 'no' teaches you something valuable. You're building resilience with each try. Keep going."
}
3. Action Suggestions for Matches
Endpoint: /api/suggestions/actions
Input:
json{
  "users": [/* same as challenge generation */]
}
MiniMax Prompt:
"Suggest 3 specific ways two people who failed at [job interviews] can practice together. Each suggestion should be practical, take under 1 hour, and build real skills."
Output:
json{
  "suggestions": [
    "Practice behavioral questions and time each other's responses",
    "Review each other's resumes and suggest 3 improvements each",
    "Research one company together and prepare questions to ask"
  ]
}
4. Onboarding Profile Enhancement
Endpoint: /api/onboarding/enhance-profile
Input:
json{
  "linkedInUrl": "https://linkedin.com/in/user",
  "OR": {
    "quizAnswers": [/* quiz responses */]
  }
}
MiniMax Prompt:
"Based on this LinkedIn profile [summary], suggest 3 potential areas where this person could help others, and 3 specific failures they might have encountered in their journey."

Database Schema (Convex.dev)
Users Collection
typescript{
  _id: Id<"users">
  username: string // "Phoenix_4521"
  category: string // "Job Search"
  goal: string
  failures: string[] // ["Failed interviews", "No responses"]
  failureDescription?: string
  severity?: number // 1-5
  points: number
  badges: string[]
  linkedInUrl?: string
  profileData?: object // from quiz or LinkedIn
  createdAt: number
  lastActive: number
}
Posts Collection
typescript{
  _id: Id<"posts">
  userId: Id<"users">
  username: string // denormalized for speed
  category: string
  content: string
  severity?: number
  encouragements: number
  encouragedBy: Id<"users">[] // prevent double-encouragement
  timestamp: number
}
Matches Collection
typescript{
  _id: Id<"matches">
  userIds: Id<"users">[]
  category: string
  chatRoomId: string
  createdAt: number
  isActive: boolean
  profilesRevealed: Id<"users">[] // who opted to reveal
}
ChatRooms Collection
typescript{
  _id: Id<"chatRooms">
  matchId: Id<"matches">
  roomName: string // "The Comeback Crew"
  messages: {
    userId: Id<"users">
    username: string
    content: string
    timestamp: number
    reactions: { emoji: string, userIds: Id<"users">[] }[]
  }[]
}
Challenges Collection
typescript{
  _id: Id<"challenges">
  chatRoomId: Id<"chatRooms">
  challenge: string
  generatedAt: number
  completedBy: Id<"users">[]
  estimatedTime?: number
}
Celebrations Collection
typescript{
  _id: Id<"celebrations">
  userId: Id<"users">
  type: "tried_again" | "badge_earned" | "milestone"
  description: string
  points: number
  timestamp: number
}
